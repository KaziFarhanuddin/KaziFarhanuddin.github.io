Chapter 3 - Greedy Algorithms

Introduction to Greedy Technique

Greedy is an algorithmic paradigm that builds up a solution piece by piece, always choosing the next piece that offers the most obvious and immediate benefit. So the problems where choosing locally optimal also leads to global solution are best fit for Greedy.

Greedy method says that the problem should be solved in stages.

xxxxxxxxxx
Algorithm Greedy(a, n)
{
    for i=1 to n do
    {
        x = Select(a)
        if Fesible(x) then
            solution = solution + x
    }
}

In this approach, the decision is taken on the basis of current available information without worrying about the effect of the current decision in future.

This approach is mainly used to solve optimization problems.

Components of Greedy Algorithm

Greedy algorithms have the following five components −

  • A candidate set − A solution is created from this set.
  • A selection function − Used to choose the best candidate to be added to the solution.
  • A feasibility function − Used to determine whether a candidate can be used to contribute to the solution.
  • An objective function − Used to assign a value to a solution or a partial solution.
  • A solution function − Used to indicate whether a complete solution has been reached.

Knapsack Problem

In knapsack problem we are given some objects and each object has a particular weight and profit, we are then given a bag with a max-weight and asked to place objects such that we get max-profit.

It is an optimization problem.

It is a maximization problem.

Here the constrain is that the total weight should be less that max-capacity of knapsack (bag).

In knapsack problem we use the ratio of profit/weight then use greedy method.

Also in knapsack problem we can divide a particular object into its rations.

void swap(int a[],int b[], int i, int j){

bool comp(int a[], int b[], int i, int j){
    return (a[i]/b[i]) > (a[j]/b[j]);
}
​
void sort(int a[], int b[], int n){
    int i, j;
    for(i=0; i<n; i++){
        j = i;
        while(j>0 &&  comp(a, b, j, j-1)){
            swap(a, b, j-1, j);
            j--;
        }
    }
}
​
double knapsackFunction(int M, int P[], int W[], int n){
    sort(P, W, n);
​
    int weight = 0;
    double profit = 0.0;
​
    for(int i=0; i<n; i++){
        if(weight+W[i] <= M){
            weight += W[i];
            profit += P[i];
        }
        else{
            int remainder = M - weight;
            profit += P[i] * remainder/W[i];
            break;
        }
    }
    return profit;
}

Minimum Spanning Tree

Spanning tree

A spanning tree is a sub-graph of a graph having all vertices but n-1 Edges (Adjacent) (n is no of vertices).

Trees have no cycle like graphs.

The no of spanning trees possible is

where :- e -> edges, v-> vertices, N-> no of cycles in graph

Prim's Algorithms (to find Minimum spanning tree)

Approach

step1: Select the min cost edge as starting point, that is two vertice

step2: Select and connect the next min-cost edge which is connected to already selected vertices.

 

Kruskal's Algorithms (to find Minimum spanning tree)

Approach

step1: Select the min cost edge as starting point, that is two vertice

step2: Select the next min-cost edge which Which dos not create a cycle.

class Edge{
public:
    int source;
    int dest;

bool condition(Edge a, Edge b){
    return a.weight < b.weight;
}
​
int getParent(int v,int *p){
    if(p[v] == v)
        return v;
    else
        getParent(p[v], p);
}
​
void kruskals(Edge*input, int n, int E){
    // Sort the input array in assinding order by weights
    sort(input, input+E, condition);
​
    //Create output array of n-1 as we only 
    //      want n-1 in our mst(minimum spanning tree)
    Edge *output = new Edge[n-1];
    
    // To track 
    int *parent = new int[n];
    for(int i=0; i<n; i++)
        parent[i] = i;
​
    // To show how many edges have been added to our mst 
    int count = 0;
    // To get current edge
    int i = 0;
​
    while(count != n-1){
        Edge current_edg = input[i];
        
        // Check if we can add the current edg in mst
        int source_parent = getParent(current_edg.source, parent); 
        int dest_parent = getParent(current_edg.dest, parent); 
​
        if(source_parent != dest_parent){
            output[count] = current_edg;
            count++;
            // We can create any one source and dest
            parent[source_parent] = dest_parent; 
        }
        i++;
    }
    for(int i=0; i<n-1; i++){
        if (output[i].source < output[i].dest)
            cout << output[i].source << " " << output[i].dest << " " << output[i].weight << endl;
        else
            cout << output[i].dest << " " << output[i].source << " " << output[i].weight << endl;
​
    }
​
}
